# Repo Orientation: Eridian

> Generated by /orient:orient. Re-run to update.

## One-line purpose
Eridian is a CLI tool that lets you speak words into a microphone, assign English labels, and build a semantic dictionary for any spoken language — inspired by the language-mapping process in *Project Hail Mary*.

## Primary language(s)
- Go (CLI, TUI, audio capture, storage)
- Python (planned — ML embedding service via FastAPI, not yet implemented)

## Pipeline / workflow stages
1. **CLI entry** — `main.go` calls `cmd.Execute()`, Cobra dispatches to the root command
2. **TUI menu** — Root command launches a Bubble Tea interactive menu for command selection
3. **Audio capture** — (planned) Go captures mic audio via portaudio
4. **ML service** — (planned) Go sends audio to a Python FastAPI sidecar over an ephemeral port
5. **Embedding & matching** — (planned) Python computes Wav2Vec 2.0 embeddings; Go stores them in SQLite and matches via nearest-centroid
6. **Storage** — (planned) One SQLite DB per language under `~/.eridian/`, Go owns all writes

- `main.go` — Entry point; delegates immediately to `cmd.Execute()` | Start here to see how the CLI boots

## Core concepts
**Meaning units** — The fundamental output of Eridian: individual words or short chunks mapped to English labels, explicitly *not* grammatically constructed sentences. This distinction drives the entire design. See `README.md` and `CLAUDE.md`.

**Centroid matching** — The planned recognition strategy: each labelled word's embedding becomes a centroid. Translation is nearest-neighbour lookup in embedding space. Adding a new word is just adding a new centroid — no retraining. Described in `CLAUDE.md` under Key Design Decisions.

**Go/Python boundary** — Go owns all state (SQLite writes, config, audio capture). Python is a stateless compute sidecar (audio in, embedding vector out) reached over an ephemeral HTTP port. This is a hard architectural rule. See `CLAUDE.md`.

**Per-language SQLite databases** — Each language gets its own `.db` file under `~/.eridian/`. Active language is tracked in `config.json`. Described in `CLAUDE.md` under Storage Layout.

## Common gotchas
- **The Python ML service doesn't exist yet.** The repo is in Phase 0 — only the Go CLI scaffolding and TUI menu are implemented. Don't look for FastAPI code or embedding logic; it's all planned.
- **Bubble Tea v2 uses `tea.View` return type**, not a plain string. See `cmd/root.go` line 117 — `View()` returns `tea.NewView(s)`, not `s` directly.
- **The TUI menu doesn't dispatch commands yet.** The `enter`/`space` handler in `Update()` toggles a selection set but doesn't actually run subcommands. This is scaffolding waiting to be wired up.
